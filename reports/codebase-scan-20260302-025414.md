### 🟠 High Priority Issues

#### [HIGH-001] Excessive Clone Usage Throughout Codebase

- **Occurrences**: 177+ instances of `.clone()`, `Arc::new()`, `Mutex::new()`
- **Files Affected**: 
  - `src/scheduler/mod.rs` (40+ clones)
  - `src/docker/run/run.rs` (30+ clones)
  - `src/docker/client.rs` (25+ clones)
  - `src/cli/mod.rs` (15+ clones)
  - `src/discord/mod.rs` (15+ clones)
- **Issue**: Extensive use of `.clone()` indicates potential ownership and borrowing issues. While sometimes necessary with async Rust, many clones could be avoided with better lifetime management or Arc-based shared state.
- **Recommendation**: 
  - Audit each `.clone()` to determine if it's necessary
  - Consider using more `& references` in function signatures
  - Use `Arc::clone()` instead of `.clone()` for Arc types (more explicit)
- **Effort**: M (4 hours)

#### [HIGH-002] Test Fixture Contains Hardcoded API Key Pattern

- **File**: `src/discord/config.rs` (lines 1070-1073)
- **Issue**: Test code expects a specific API key value from a test fixture:
  ```rust
  llm.api_key_env,
  "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",
  ```
- **Risk**: If this test fixture is not properly managed, sensitive keys could be committed
- **Recommendation**: 
  - Ensure test fixtures use placeholder/environment-based values
  - Add `.gitignore` entry for any switchboard.toml that might contain real keys
  - Consider using mock values in tests rather than reading from actual config files
- **Effort**: S (30 minutes)

#### [HIGH-003] Magic Numbers in Code

- **Files**: Multiple locations
- **Issue**: Hardcoded numeric values without constants:
  - Timeout values (30, 60, 300 seconds)
  - Queue sizes
  - Sleep durations
  - Buffer sizes
- **Recommendation**: Extract to named constants:
  ```rust
  const DEFAULT_TIMEOUT_SECONDS: u64 = 30;
  const DEFAULT_QUEUE_SIZE: usize = 3;
  const DEFAULT_RETRY_COUNT: u32 = 3;
  ```
- **Effort**: M (2 hours)
